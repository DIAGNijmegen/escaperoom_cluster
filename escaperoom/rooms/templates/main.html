SOL2 Terminal

Welcome to the SOL2 Terminal. Here you can access the SOL2 cluster and manage the nodes.

Prepare for Your Escape:

Before you embark on this quest, ensure that you are equipped with the following:
  • Fully charged laptop with at least 2 hours of battery life.
  • Spare power extension cable, if available.
  • Github and SOL2 accounts.

Escape notes:

Today's mission is not just about escape, but about learning and collaboration in a safe environment. Your goal is to (virtually) escape from a cold, locked SOL server room by solving a series of tasks set by the SOL administrators. For this you will need to build a Docker image and push it to the registry. Run an interactive job on the cluster and execute a Python script using a normal SBATCH command. Good luck!

Each team member is expected to work on the tasks independently on their own laptop. Progress to the next task is only possible when all team members have completed the current task. In each team, there's an expert on SOL2, but remember: the expert should guide and assist rather than solve the tasks themselves. Experts are there to ensure everyone moves at their own pace and learns effectively. Cooperation between different teams is encouraged.

Prizes await teams that not only complete the tasks but also excel in creating a positive and effective learning atmosphere.

Find your name and team number in the 'ImageTeams.xlsx' on Teams or contact the organizer if you're not yet assigned.

Let's learn, collaborate, and escape together!

Special Message for Team Experts, before you start

Team Experts are required to take the following steps:

  1. Create a private GitHub repository within DIAGNijmegen, give it a nice teamname.
  2. Invite all members of your team to the repository and grant them admin rights.
  3. Invite Daan and Clement to the repository and grant them admin rights as well.
  4. Let every user push a file called: 'logged_in.txt' in a teammember specific folder, this will be their folder during this Escape
  5. After completing these steps, create an Issue in the GitHub repository and mention Daan and Clement in it.
  6. You will receive a key in the Issue.
